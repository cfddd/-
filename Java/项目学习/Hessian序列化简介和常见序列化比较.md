## 简介
Hessian 本质就是一种 **对象序列化协议**，而且是 **二进制格式** 的序列化协议
- **对象序列化**： Hessian 能把 Java 对象（甚至是复杂对象、集合、嵌套对象）序列化成二进制流。序列化结果里包含了 **类信息（class 名字、字段名）+ 字段值**。反序列化时，它就能按照协议把二进制还原成对象。
- **二进制**：相比 JSON、XML 这些文本序列化，Hessian 是二进制，体积更小、解析更快，但缺点是 **不可读**，调试时需要工具才能看懂。
- **跨语言**： Hessian 不是只给 Java 用的，它有 PHP、Python、C# 等语言实现。
## 比较
| 特性             | JSON                  | Hessian                  | Protobuf               |
| -------------- | --------------------- | ------------------------ | ---------------------- |
| 数据格式           | 文本（UTF-8）             | 二进制                      | 二进制                    |
| 可读性            | ✅ 高，可直接查看             | ❌ 低                      | ❌ 低                    |
| 支持复杂对象         | ✅ 支持嵌套对象              | ✅ 支持 Java Bean、嵌套对象      | ✅ 支持嵌套对象               |
| 序列化体积          | 较大                    | 中等                       | 小                      |
| 序列化/反序列化**性能** | 中                     | 中等                       | 快                      |
| 跨语言            | ✅ 各语言都支持              | ✅ Java、Python、C# 等       | ✅ 各语言官方支持              |
| schema         | 无                     | 可无（依赖类名和字段名）             | 必须 `.proto` 文件定义       |
| 版本兼容性          | 弱（字段增删需处理）            | 弱（类或字段改动需小心）             | 强（遵守字段号规则）             |
| 典型场景           | REST API、日志、配置、业务数据交换 | 分布式 RPC（Dubbo）、监控埋点（CAT） | 微服务 RPC（gRPC）、高性能跨语言通信 |
