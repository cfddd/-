顾名思义，是登录鉴权用的工具
> `StpUtil` 本质上是 **Sa-Token 的门面工具类**，它并不是 Hutool 的东西
## java登录方案对比

| 框架/方式                    | 特点                                                                             | 适用场景                            |
| ------------------------ | ------------------------------------------------------------------------------ | ------------------------------- |
| **Sa-Token (`StpUtil`)** | 简单、轻量、国内社区活跃。支持 **登录鉴权、角色权限、Session、分布式会话、踢人下线** 等。API 类似 Hutool，学习曲线低。        | 中小型项目、微服务网关鉴权、想快速上手。            |
| **Spring Security**      | 功能极其完善，支持 **RBAC、OAuth2、JWT、Method Security**，但学习成本高、配置复杂。                     | 大型项目、需要细粒度安全控制、合规要求高。           |
| **Shiro**                | 老牌框架，支持认证、授权、加密、Session。API 简洁，但 **更新慢、生态弱化**。                                 | 传统项目，或者 Spring Boot 早期集成。       |
| **JWT（纯 Token 方案）**      | 前后端分离常见做法，token 自包含（用户信息 + 过期时间），无状态，不依赖 session/redis。缺点：**不可强制踢人**，一旦签发无法撤销。 | 移动端、无状态 REST API，允许 token 自然过期。 |
## 最佳实践
**全局拦截器**：
```java
@Configuration
public class SaTokenConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new SaInterceptor(handler -> {
            // 拦截规则：比如所有 /api/** 都要登录
            SaRouter.match("/api/**", r -> StpUtil.checkLogin());
        })).addPathPatterns("/**");
    }
}
```
**不使用本地缓存，使用redis作为session**：
```java
// 会自动注入 SaTokenDao
@Bean
public SaTokenDao saTokenDao(StringRedisTemplate redisTemplate) {
    SaTokenDaoRedisJackson dao = new SaTokenDaoRedisJackson(redisTemplate);
    dao.setSafeKeyPrefix("satoken:"); // Redis key 前缀
    return dao;
}
```
