## Rlock概述
Redisson 是一个非常强大的 **Java Redis 客户端**，它不仅仅是一个简单的 Redis 客户端，还封装了许多高级功能，这里主要学习其中**分布式锁（RLock）** 的部分
## 分布式锁使用示例
**环绕注解**
```java
@Aspect  
@Component  
public class RedisLockAspect {  
  
    @Autowired  
    private RedissonClient redissonClient;  
  
    private static final String REDISSON_LOCK_PREFIX = "redisson_lock:";  
  
    @Around("@annotation(redisLock)")  
    public Object around(ProceedingJoinPoint joinPoint, RedisLock redisLock) throws Throwable {  
       String spel = redisLock.key();  
       String lockName = redisLock.lockName();  
  
       RLock rLock = redissonClient.getLock(getRedisKey(joinPoint,lockName,spel));  
  
       rLock.lock(redisLock.expire(),redisLock.timeUnit());  
  
       Object result = null;  
       try {  
          //执行方法  
          result = joinPoint.proceed();  
  
       } finally {  
          rLock.unlock();  
       }  
       return result;  
    }  
  
    /**  
     * 将spel表达式转换为字符串  
     * @param joinPoint 切点  
     * @return redisKey  
     */    private String getRedisKey(ProceedingJoinPoint joinPoint,String lockName,String spel) {  
       Signature signature = joinPoint.getSignature();  
       MethodSignature methodSignature = (MethodSignature) signature;  
       Method targetMethod = methodSignature.getMethod();  
       Object target = joinPoint.getTarget();  
       Object[] arguments = joinPoint.getArgs();  
       return REDISSON_LOCK_PREFIX + lockName + StrUtil.COLON + SpelUtil.parse(target,spel, targetMethod, arguments);  
    }  
}
```
SpEL（Spring Expression Language）是Spring框架提供的一个强大的表达式语言，用于在运行时查询和操作对象图。

- **字面量表达式**：`'Hello World'`, `123`, `true`
- **变量引用**：`#variableName`
- **属性访问**：`object.property`
- **方法调用**：`object.method()`
- **数组和集合访问**：`array[0]`, `list[0]`
- **操作符**：
	- 关系操作符：`==`, `!=`, `<`, `>`, `<=`, `>=`
	- 逻辑操作符：`and`, `or`, `not`
	- 数学操作符：`+`, `-`, `*`,`/`

SpelUtil工具类会将表达式与实际的方法参数、对象上下文相结合，计算出表达式的实际值。

**切点**
```java
  
/**  
 * 使用redis进行分布式锁  
 * @author lanhai  
 */@Target(ElementType.METHOD)  
@Retention(RetentionPolicy.RUNTIME)  
@Documented  
public @interface RedisLock {  
  
    /**  
     * redis锁 名字  
     */  
    String lockName() default "";  
  
    /**  
     * redis锁 key 支持spel表达式  
     */  
    String key() default "";  
  
    /**  
     * 过期秒数,默认为5毫秒  
     *  
     * @return 轮询锁的时间  
     */  
    int expire() default 5000;  
  
    /**  
     * 超时时间单位  
     *  
     * @return 秒  
     */  
    TimeUnit timeUnit() default TimeUnit.MILLISECONDS;  
}
```
## 可重入锁-实现基本流程
Redisson 用的是 **Lua 脚本 + Hash 结构** 来实现可重入。

Hash：
- key = 锁名
- field = 线程标识（UUID + threadId）
- value = 重入次数 (int)
### 第一次加锁
```java
if not exists(KEYS[1]) then
    hset(KEYS[1], threadId, 1)    -- 设置重入计数为 1
    pexpire(KEYS[1], leaseTime)   -- 设置过期时间
    return nil
elseif hexists(KEYS[1], threadId) then
    hincrby(KEYS[1], threadId, 1) -- 重入，计数 +1
    pexpire(KEYS[1], leaseTime)   -- 续期
    return nil
else
    return pttl(KEYS[1])          -- 锁被别人占了，返回剩余过期时间
end
```

- **空锁 → 当前线程写入，计数 = 1**
- **自己已持有 → 计数 +1**
- **别人持有 → 返回剩余 TTL，等待或失败**

**tryLock(waitTime, leaseTime)**

- Redisson 会在 waitTime 内循环执行上面的 Lua 脚本。
- 如果别人占用锁，线程挂起 + 等待剩余时间 + 重试。
- 如果 waitTime 到了还没抢到 → 返回失败。

### **自动续期（WatchDog）**

WatchDog 每隔 leaseTime / 3 时间检查一下，如果当前线程还在持有锁 → 调用 pexpire() 续期。防止业务执行超时导致锁提前过期。默认续期时间：30s
### 释放锁 (unlock) 流程
```lua
if not exists(KEYS[1]) then
    return nil
end
if not hexists(KEYS[1], threadId) then
    return nil
end
local counter = hincrby(KEYS[1], threadId, -1) -- 计数 -1
if counter > 0 then
    pexpire(KEYS[1], leaseTime) -- 还在持有，刷新过期时间
    return 0
else
    del(KEYS[1])                -- 计数归零，真正释放锁
    return 1
end
```
**重入计数 > 0** → 只减计数，不释放

**重入计数 = 0** → 删除锁

### 小结
```sh
lock():
    ├─ KEYS[1] 不存在 → 设置计数=1，成功
    ├─ KEYS[1] 存在 & threadId 相同 → 重入计数+1，成功
    └─ KEYS[1] 存在 & threadId 不同 → 返回 TTL，失败

unlock():
    ├─ threadId 相同 → 计数-1
    │    ├─ >0 → 保持锁
    │    └─ =0 → 删除 key，释放锁
    └─ threadId 不同 → 不操作（防止误删）
```
