
### **1. 核心数据结构**

#### **`RingBuffer<T>`**
```java
import com.lmax.disruptor.RingBuffer;
```

- **作用**：环形缓冲区，Disruptor的核心数据结构
- **特点**：预分配数组，固定大小（必须是2的幂次方）
- **功能**：存储事件对象，提供序列号管理

#### **Sequence**
```java
import com.lmax.disruptor.Sequence;
```
- **作用**：序列号管理，追踪生产者和消费者位置
- **特点**：volatile long值，带缓存行填充
- **功能**：无锁的位置标记和同步机制

### **2. 事件相关类**

#### **Event（事件类）**
```java
// 用户自定义，如您的 OrderEvent
public class OrderEvent {
    private long orderId;
    private String productName;
    // ...
}
```
- **作用**：在RingBuffer中传递的数据载体
- **特点**：可重用对象，避免频繁GC

#### **`EventFactory<T>`**
```java
import com.lmax.disruptor.EventFactory;

public class OrderEventFactory implements EventFactory<OrderEvent> {
    public OrderEvent newInstance() {
        return new OrderEvent();
    }
}
```
- **作用**：创建事件实例的工厂
- **功能**：预分配阶段创建所有事件对象

#### **`EventHandler<T>`**
```java
import com.lmax.disruptor.EventHandler;

public class OrderEventHandler implements EventHandler<OrderEvent> {
    public void onEvent(OrderEvent event, long sequence, boolean endOfBatch) {
        // 处理事件逻辑
    }
}
```
- **作用**：事件处理器，定义如何处理事件
- **功能**：消费者的业务逻辑入口

### **3. 生产者相关**

#### **`Disruptor<T>`**
```java
import com.lmax.disruptor.dsl.Disruptor;

Disruptor<OrderEvent> disruptor = new Disruptor<>(
    factory, bufferSize, threadFactory, 
    ProducerType.SINGLE, waitStrategy
);
```
- **作用**：Disruptor的主入口类（DSL）
- **功能**：配置和管理整个Disruptor系统

#### **ProducerType**
```java
import com.lmax.disruptor.dsl.ProducerType;

ProducerType.SINGLE  // 单生产者（性能更高）
ProducerType.MULTI   // 多生产者
```
- **作用**：定义生产者模式
- **影响**：决定序列号分配策略

#### **Sequencer（序列器）**
```java
// 内部类，不直接使用，但很重要
SingleProducerSequencer    // 单生产者序列器
MultiProducerSequencer     // 多生产者序列器
```
- **作用**：管理序列号分配和发布
- **功能**：生产者的核心控制组件

### **4. 等待策略**

#### **WaitStrategy（等待策略接口）**
```java
// 各种等待策略实现：

BlockingWaitStrategy     // 阻塞等待，CPU友好
YieldingWaitStrategy     // 自旋+yield，低延迟
SleepingWaitStrategy     // 渐进式睡眠等待
BusySpinWaitStrategy     // 纯自旋，最低延迟
TimeoutBlockingWaitStrategy // 带超时的阻塞等待
```
- **作用**：定义消费者如何等待新事件
- **选择**：平衡延迟、吞吐量、CPU使用率

## 📊 类的层次结构总结

```
Disruptor<T>                    // 主入口，DSL配置
├── RingBuffer<T>              // 核心数据结构
│   ├── Sequencer             // 序列管理
│   └── Event[]               // 事件数组
├── EventProcessor            // 事件处理器包装
│   ├── BatchEventProcessor   // 批处理器
│   └── WorkProcessor         // 工作处理器
├── WaitStrategy             // 等待策略
├── EventHandler<T>          // 业务处理逻辑
├── EventFactory<T>          // 事件工厂
└── ExceptionHandler<T>      // 异常处理
```

## 🎯 核心交互流程

```
1. EventFactory 创建事件对象填充到 RingBuffer
2. Producer 通过 Sequencer 获取序列号
3. Producer 设置事件数据并 publish
4. EventProcessor 通过 WaitStrategy 等待新事件
5. EventHandler 处理具体业务逻辑
6. SequenceBarrier 协调多个处理器的依赖关系
```

这些类协同工作，构成了Disruptor高性能、低延迟的事件处理框架！
