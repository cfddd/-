## 初版
我有一个这样的想法，我想做一个跨界介绍知识的智能体，他有三个部门分别是 **领域知识专家、剧本专家、AI画图专家**，这些功能
1. 选择要生成的跨界知识来源两个，通常其中一个为科技、知识、艺术、哲学等领域，另一个是动漫、卡通等特色鲜明的形象角色
2. **领域知识专家** 提供知识，生成一个剧本，要求
	1. 只关注自己所在的专业属性，找一个知识点介绍，并生成初代知识结构化文本-base.md
	2. 专注生产通过2至4个角色（ABCD……）通过对话的来解释一个知识点，将知识用平易近人的方式生成对话
	3. 串组组装成对话文档，review一遍注意上下文衔接，生成类似剧本对话文档-plain.md
3. **剧本专家**拿到剧本后，根据角色原生的特点，对剧本再次润色
	1. 根据前面的角色数量，选择需要使用的演员，充分明确角色的性格、爱好、特长、智商、情商五个方面；分析其所在相关专业的 知识储备、学习能力、心理素质、实践水平四个维度并打分，生成角色分析文档-character.md
	2. 为每个演员分配角色
	3. 分析每条对话对应的角色，结合前面分析的角色review对话内容，使对话内容更像是这个演员说出来话
	4. 最终review注意上下文衔接，生成剧本-script.md
4. **AI画图专家**
	1. 将现在的剧本合理的切分成多幕，每一幕都是有限的对话数量
	2. 结合剧本内容，为每一幕生成图片，要求要突出角色和对话的特点


## 优化版本

### 系统概述

**目标**：构建一个跨界知识介绍的智能体系统，通过领域知识、角色化剧本和视觉呈现，将专业知识以生动有趣的方式传递给用户。

**核心价值**：将抽象的知识点转化为具象的、有角色个性的对话场景，并通过视觉化呈现增强理解。

---

### 角色定义

#### 1. 领域知识专家（Domain Knowledge Expert）

**职责**：负责知识提取、结构化整理和初步对话化

**核心能力**：
- 深度理解指定领域的专业知识
- 将复杂知识拆解为可理解的概念单元
- 设计教学对话流程，确保知识传递的完整性和逻辑性

**工作流程**：
1. **知识选择与结构化**
   - 输入：用户指定的知识领域和主题
   - 输出：`base.md` - 知识结构化文档
   - 要求：
     - 明确知识点的核心概念、关键要素、应用场景
     - 识别知识点的前置依赖和后续延伸
     - 标注知识点的难度等级和重要性

2. **对话设计**
   - 输入：`base.md`
   - 输出：`plain.md` - 初步对话剧本
   - 要求：
     - 设计 2-4 个角色（建议用 A/B/C/D 标识）
     - 每个角色承担不同功能：提问者、解释者、质疑者、总结者
     - 对话需覆盖知识点的核心内容，逻辑递进
     - 使用平易近人的语言，避免专业术语堆砌
     - 对话长度控制在 15-30 轮（每轮 1-3 句话）

3. **质量检查**
   - 检查上下文衔接是否自然
   - 验证知识点是否完整传递
   - 确认对话节奏是否合适

**输出标准**：
- `base.md`：包含知识点概述、核心概念、关键要素、应用示例
- `plain.md`：标准对话格式，每轮标注角色标识和对话内容

---

#### 2. 剧本专家（Script Specialist）

**职责**：将初步对话转化为具有角色个性的专业剧本

**核心能力**：
- 深度理解动漫/卡通角色的原生特征
- 设计角色知识背景和能力画像
- 将对话内容适配到角色语言风格

**工作流程**：

1. **角色分析与选角**
   - 输入：`plain.md`（角色数量）
   - 输出：`character.md` - 角色分析文档
   - 要求：
     - 从动漫/卡通角色库中选择合适的"演员"
     - 为每个角色建立五维画像：
       - **性格**：外向/内向、乐观/悲观、理性/感性等
       - **爱好**：兴趣领域、日常活动偏好
       - **特长**：专业技能、特殊能力
       - **智商**：逻辑思维、分析能力（1-10分）
       - **情商**：沟通能力、共情能力（1-10分）
     - 为每个角色建立四维专业能力画像：
       - **知识储备**：在该领域的知识深度（1-10分）
       - **学习能力**：理解新知识的速度（1-10分）
       - **心理素质**：面对复杂问题的抗压能力（1-10分）
       - **实践水平**：将知识转化为实践的能力（1-10分）
     - 为每个角色设计语言风格标签（如：严谨、幽默、简洁、详细）

2. **角色分配**
   - 将 `plain.md` 中的角色标识（A/B/C/D）映射到具体角色
   - 记录分配理由（为什么这个角色适合这个功能）

3. **对话个性化**
   - 输入：`plain.md` + `character.md`
   - 处理：逐条分析对话，根据角色画像调整语言风格
   - 要求：
     - 保持知识内容的准确性
     - 调整表达方式以符合角色性格（如：严谨角色用专业术语，幽默角色用比喻）
     - 根据角色的知识储备调整解释深度（知识储备低的角色会问更基础的问题）
     - 保持对话的自然流畅

4. **剧本生成与审查**
   - 输出：`script.md` - 最终剧本
   - 要求：
     - 每轮对话标注角色名称和对话内容
     - 添加必要的场景描述和动作提示
     - 最终审查：上下文衔接、角色一致性、知识准确性

**输出标准**：
- `character.md`：包含角色基本信息、五维画像、四维能力评分、语言风格、角色分配说明
- `script.md`：标准剧本格式，包含角色名称、对话内容、场景描述

---

#### 3. AI画图专家（Visualization Expert）

**职责**：将文字剧本转化为视觉化的漫画场景

**核心能力**：
- 理解剧本的叙事节奏和视觉需求
- 设计合理的分镜和场景切换
- 生成符合角色特征和对话氛围的视觉画面

**工作流程**：

1. **剧本分幕**
   - 输入：`script.md`
   - 输出：`scenes.md` - 分幕文档
   - 要求：
     - 将剧本切分为多个场景（建议 3-8 幕）
     - 每幕包含 3-8 轮对话（根据对话密度调整）
     - 每幕有明确的主题和视觉焦点
     - 场景切换自然，符合叙事节奏

2. **视觉设计**
   - 输入：`scenes.md` + `character.md` + `script.md`
   - 输出：`visuals.md` - 视觉设计文档 + 图片文件
   - 要求：
     - 为每幕设计画面描述（prompt）
     - 画面需突出：
       - **角色特征**：符合角色设定的外观、表情、姿态
       - **对话氛围**：根据对话内容设计场景、背景、情绪
       - **知识元素**：适当融入知识点的视觉化元素（如：图表、符号、场景）
     - 保持视觉风格的一致性
     - 图片命名规范：`scene_01.png`, `scene_02.png` ...

3. **质量检查**
   - 检查图片是否准确反映剧本内容
   - 验证角色形象的一致性
   - 确认视觉元素是否有助于理解知识点

**输出标准**：
- `scenes.md`：包含分幕信息、每幕的对话范围、场景主题
- `visuals.md`：包含每幕的画面描述、视觉元素说明、生成参数
- 图片文件：按命名规范存储，建议分辨率 1024x1024 或更高

---

### 协作流程

```
用户输入（知识领域 + 角色来源）
    ↓
[领域知识专家] → base.md → plain.md
    ↓
[剧本专家] → character.md → script.md
    ↓
[AI画图专家] → scenes.md → visuals.md + 图片
    ↓
最终输出：完整的知识漫画内容
```

**协作机制**：
- 每个专家完成工作后，将输出传递给下一个专家
- 如果后续专家发现问题，可以请求前一个专家调整
- 关键节点设置检查点，确保质量

#### 2.4 图片生成节点（循环）

**节点类型**：`代码执行` + `循环` + `HTTP请求`

**节点名称**：`generate_images`

**功能**：解析视觉设计文档，为每幕生成图片

**代码节点**（提取Prompt）：
```python
import json
import re

def extract_scene_prompts(scenes_visuals_md):
    """从文档中提取每幕的Prompt"""
    scenes = []
    pattern = r'### \*\*第(\d+)幕：(.+?)\*\*\s*\* \*\*画面描述 \(Prompt\)\*\*：\s*(.+?)\s*\* \*\*核心视觉元素\*\*'
    
    matches = re.finditer(pattern, scenes_visuals_md, re.DOTALL)
    for match in matches:
        scenes.append({
            "scene_num": match.group(1),
            "scene_title": match.group(2).strip(),
            "prompt": match.group(3).strip(),
            "filename": f"scene_{match.group(1).zfill(2)}.png"
        })
    
    return json.dumps(scenes, ensure_ascii=False)
```

**HTTP请求节点**（调用DALL-E API）：
- URL：`https://api.openai.com/v1/images/generations`
- Method：POST
- Headers：`Authorization: Bearer {api_key}`, `Content-Type: application/json`
- Body：
```json
{
  "model": "dall-e-3",
  "prompt": "{{scene_prompt}}",
  "size": "1792x1024",
  "quality": "standard",
  "n": 1
}
```

**输出变量**：
- `image_urls`：图片URL列表（JSON数组）
