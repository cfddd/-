## 对比

| 协议              | 概述                                                                                                                                                                                                                                     | ✅优点                                                                                                                                                                                                                                                                                                                                                                                                                                         | ❌ 缺点                                                                                                                                                                                                                                                                                                                                                                             |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **HTTP协议**      | 简单的说就是一种不支持长久链接的状态的一种通讯协议，执行后从后端返回数据就结束了。                                                                                                                                                                                              | 协议成熟、稳定、通用性强；<br><br>支持缓存、代理、中间件，兼容性好；<br><br>与浏览器和调试工具高度兼容（如 Chrome DevTools、Postman）；<br><br>防火墙和企业网络环境下表现良好                                                                                                                                                                                                                                                                                                                              | 不支持服务器主动推送（需轮询或使用 SSE）；<br><br>每次请求都需重新建立连接，通信效率低；<br><br>请求头部开销大，不适合高频通信；<br><br>不适用于高实时性需求场景（如聊天室、在线游戏）                                                                                                                                                                                                                                                                        |
| **WebSocket协议** | 请求连接后浏览器和客户端会一直保持链接。由于一直在保持链接，那么后端有数据更新后前端浏览器就可以实时获取到。                                                                                                                                                                                 | 支持 双向通信，适合实时交互应用；<br><br>一次握手后维持长连接，通信效率高；<br><br>支持服务器主动推送，适合消息、行情推送等场景；<br><br>支持传输文本和二进制数据，数据开销小                                                                                                                                                                                                                                                                                                                                         | 实现较复杂，需要处理心跳、断线重连、连接状态等；<br><br>在防火墙、代理服务器环境下兼容性较差；<br><br>长连接持续占用服务器资源，连接数过多时需考虑性能优化；<br><br>调试工具支持不如 HTTP 成熟                                                                                                                                                                                                                                                                   |
| **SSE协议**       | SSE协议请求的时候可以和类似HTTP一样进行一次请求，接收数据的时候可以和WebSocket一样长时间接收<br><br>1）浏览器通过 JavaScript 创建一个 `EventSource` 对象；<br><br>2）`EventSource` 会向服务端发起一个 HTTP 请求（默认是 GET）；<br><br>3）服务端保持连接不断开，**持续不断地通过文本流格式向客户端推送数据**；<br><br>4）客户端注册回调函数，接收并处理这些事件。 | SSE协议请求的时候可以和类似HTTP一样进行一次请求，接收数据的时候和WebSocket一样长时间接收；<br><br>1. **简单易用**：SSE 实现简单，客户端和服务器端的代码都比较简单。服务器只需要在响应头中设置 `Content-Type: text/event-stream`，并开始发送事件数据。<br>    <br>2. **兼容性好**：SSE 兼容所有支持 HTTP 的浏览器，不需要额外的插件或复杂的配置。<br>    <br>3. **轻量级**：相比于 WebSocket，SSE 使用的是标准的 HTTP 协议，不需要额外的协议栈，因此开销较小。<br>    <br>4. **自动重连**：浏览器原生支持自动重连机制，当连接断开时，浏览器会自动尝试重新连接。<br>    <br>5. **跨域支持**：通过设置 CORS（Cross-Origin Resource Sharing），SSE 可以支持跨域通信。 | 1. **只能由服务器向客户端单向传输**：SSE 仅支持服务器向客户端的单向数据传输，如果需要双向通信，则需要使用 WebSocket 或其他技术。<br>    <br>2. **不支持二进制数据**：SSE 仅支持文本数据传输，不支持二进制数据的传输。对于需要传输图片、视频等二进制数据的应用场景，需要采用其他技术如 WebSocket。<br>    <br>3. **延迟和带宽限制**：由于 SSE 基于 HTTP 协议，它依赖于 HTTP 的持久连接。在某些网络环境或服务器配置下，可能会有连接超时或频繁重连的问题。<br>    <br>4. **安全性问题**：虽然可以通过 CORS 控制跨域访问，但 SSE 主要依赖于 HTTP 头部进行控制，对于更复杂的安全需求（如认证和授权），可能需要额外的服务器端逻辑处理。 |
