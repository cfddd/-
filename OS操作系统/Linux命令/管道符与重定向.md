### 区别概览
- **作用对象不同**  
  - **管道 `|`**: 把前一个命令的“标准输出”作为后一个命令的“标准输入”。命令对命令。  
  - **重定向 `>`/`>>`**: 把命令的“标准输出”写入文件（覆盖/追加）。命令对文件。

- **数据流方向**  
  - **`|`**: cmd1 stdout → cmd2 stdin（内存/内核缓冲区中传递，不落盘）。  
  - **`>`**: cmd stdout → file（写入磁盘）。

- **是否落盘**  
  - **`|`**: 一般不落盘。  
  - **`>`/`>>`**: 一定落盘（覆盖/追加）。

- **典型用法**  
  - **`|`**: 组合多个命令做流水线处理。  
    - 例：`cat access.log | grep 500 | wc -l`  
  - **`>`**: 保存结果到文件；`>>` 追加。  
    - 例：`grep 500 access.log > errors.txt`、`date >> log.txt`

- **错误输出处理**  
  - 默认只有“标准输出”参与。若要把“标准错误”一起处理：  
    - 管道：`cmd1 2>&1 | cmd2`  
    - 重定向：`cmd 1>out.txt 2>err.txt` 或 `cmd &>all.txt`（bash/zsh）

- **并行与时机**  
  - **`|`**: 两个命令通常并发执行，上游输出被下游实时消费。  
  - **`>`**: 只有一个命令执行，结束时文件已写好。

- **覆盖 vs 追加**  
  - **`>`** 覆盖写；**`>>`** 追加写。

- **与 `<` 的关系**  
  - **`<`** 是输入重定向：从文件作为命令的标准输入。  
    - 例：`sort < data.txt`

### 快速对照
- 想把结果继续交给下一个命令处理 → 用 `|`  
- 想把结果保存到文件（覆盖/追加） → 用 `>`/`>>`
